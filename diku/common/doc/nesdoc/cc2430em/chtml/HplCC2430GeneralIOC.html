Component: HplCC2430GeneralIOC

module HplCC2430GeneralIOC

Provide cc2430 specific register maps Absolute addressing in Keil is possible in a number of ways: Standard C (uint8_t xdata*) addr Using sfr types/storrage class specifiers sfr x = addr Using the at modifier uint8_t x at addr Regarding multibyte values, it seems that ChipCon has chosen the follwing semantics that handles latching one or more bytes when reading and writing: When reading: the _low_ byte must be read first and the high-byte is latched for glitch free 16-bit operations When writing: the low byte must be written first and the value does not take effect before the high byte is written
